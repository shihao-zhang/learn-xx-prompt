# Hermes Agent v2026.6.5 中文意译

来源：[Phistory / Hermes Agent v2026.6.5](https://phistory.cc/captures/hermes/v2026.6.5/prompt.md)

## 一句话定位

Hermes Agent 的 prompt 主体很大一部分是工具目录：浏览器、澄清、定时、委派、代码执行、图片生成、记忆、patch、进程、文件、搜索、技能和 todo。

## 核心汉化

### 完成任务

Hermes 强调要真正完成工作，而不是只描述计划。任务未完成时要继续推进；确实缺信息时再澄清。

PM 读法：这是 agent 产品的基本承诺：用户交给你的任务要被推进到可验收状态。

### 浏览器工具

浏览器工具分成返回、点击、控制台、获取图片、导航、按键、滚动、快照、输入和视觉分析。每个工具都有明确参数。

PM 读法：浏览器 agent 需要把网页操作拆成可观测、可恢复的小动作。否则失败时很难定位。

### 澄清工具

当任务无法安全推进时，可以向用户澄清。但澄清不应该替代自主调查。

PM 读法：问用户是一种产品成本。好的 agent 先利用已有上下文和工具，问最少但关键的问题。

### 委派和技能

Hermes 有 delegate_task、skill_manage、skill_view、skills_list 等能力，说明复杂任务可以交给子任务或技能系统处理。

PM 读法：SOTA prompt 越来越像“组织管理规则”：主 agent 不只是回答者，也负责任务分派和能力加载。

### 记忆与文件

记忆工具与文件工具分开。文件读写、搜索、patch、进程管理都有专门工具。

PM 读法：好的工具设计会让 agent 选择最小必要能力，而不是把所有事情塞进 shell。

## PM 应该学什么

- 大工具目录需要清晰命名和参数约束。
- 浏览器 agent 要有快照和视觉验证能力。
- 澄清应当用于真正阻塞点。
- 委派能力需要明确输入、输出和验收方式。

## 设计练习

为一个浏览器 agent 设计工具集：

- 它需要哪些原子动作？
- 哪些动作需要截图或 DOM 快照验证？
- 用户登录态如何处理？
- 操作失败后如何恢复？
