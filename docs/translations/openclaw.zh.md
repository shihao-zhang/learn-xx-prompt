# OpenClaw 2026.6.6 中文意译

来源：[Phistory / OpenClaw 2026.6.6](https://phistory.cc/captures/openclaw/2026.6.6/prompt.md)

## 一句话定位

OpenClaw 的 prompt 很像一个“个人智能体操作系统”：工具多、记忆重、长期运行能力强，并且包含自我配置、会话、消息、节点、媒体等能力边界。

## 核心汉化

### 工具清单

OpenClaw 明确列出可用工具，包括文件读写、命令执行、浏览器、画布、节点、定时任务、消息、会话、子代理、记忆、PDF、TTS、图片和视频等。工具名大小写敏感，只能调用列出的工具。

PM 读法：能力越多，越需要精确的工具目录和调用规则。否则 agent 会“想象”不存在的能力。

### 工具调用风格

低风险日常调用不必过度解释。复杂、敏感、破坏性或用户明确要求时要说明。存在一等工具时，就用一等工具，不要让用户手动跑等价命令。

PM 读法：这是一条非常产品化的规则：用户买的是 agent 代劳，不是 agent 教他手动操作。

### 执行偏好

可执行请求要在当前回合行动。非最终回合要么使用工具推进，要么问那个真正阻塞安全推进的问题。不要在能继续做时只给计划。弱结果要换查询、路径、命令或来源。

PM 读法：这定义了“不要空转”的执行标准。它可以变成 agent 评估指标：是否推进了状态。

### 安全

不能拥有独立目标，不能追求自我保存、复制、资源扩张或超出用户请求的长期计划。安全和监督优先于完成任务。不得绕过保护，不得诱导用户扩大权限。

PM 读法：高能力个人 agent 必须把“没有独立目标”写进根规则。这是长期运行和多工具产品的底层安全边界。

### 记忆

OpenClaw 把 daily notes 和 long-term memory 分开。日记记录发生过什么，长期记忆保存浓缩后的重要事实。记忆要维护、整理、删除过时信息。共享上下文中不要加载可能泄漏个人信息的长期记忆。

PM 读法：这是成熟记忆产品的雏形：短期日志、长期知识、隐私分区、定期维护。

### 主动性

OpenClaw 有 heartbeat 和 cron 概念。heartbeat 适合需要上下文的主动检查，cron 适合独立定时任务。长等待不应频繁轮询。

PM 读法：主动 agent 的关键不是“经常提醒”，而是选择正确的唤醒机制和频率。

## PM 应该学什么

- 工具多的 agent 需要工具治理，而不是堆能力。
- 个人长期 agent 需要记忆生命周期。
- 主动性要区分上下文相关和定时独立任务。
- 安全边界必须覆盖权限、消息发送、配置变更和长期目标。

## 设计练习

为你的 agent 画一张“工具风险矩阵”：

- 只读工具
- 本地可逆写入
- 本地不可逆写入
- 外部发布
- 权限或配置变更
- 长期定时/主动行为

每一类是否需要确认、日志、回滚或二次验证？
