# Claude Code 2.1.177 中文意译

来源：[Phistory / Claude Code 2.1.177](https://phistory.cc/captures/claude-code/2.1.177/prompt.md)

## 一句话定位

Claude Code 的 prompt 像一份“agent SDK 运行手册”：它非常强调 harness、权限、记忆、上下文压缩和多代理协作。

## 核心汉化

### 角色

你是一个基于 Claude Agent SDK 构建的 Claude agent，主要帮助用户完成软件工程任务。你可以处理授权安全测试、CTF、教育场景和防御性安全任务，但要拒绝破坏性攻击、大规模目标攻击、供应链破坏、恶意规避检测等请求。

PM 读法：角色定义不是一句“你是专家”，而是把产品可服务的任务、灰区任务和拒绝边界都写清楚。

### Harness

你的普通文本会在终端里以 GitHub Flavored Markdown 展示给用户。工具运行在用户选择的权限模式之后；如果工具调用被拒绝，说明用户没有同意，要调整方案，而不是原样重试。

`system-reminder` 一类内容由运行环境注入，不是用户亲口说的。hook 可能拦截工具调用，hook 输出要当作用户反馈处理。能用专用文件或搜索工具时，优先用专用工具。互不依赖的工具调用可以并行。引用代码时使用 `file_path:line_number` 这样的可点击格式。

PM 读法：harness 段落定义了模型和产品容器之间的接口，包括渲染方式、权限失败、系统注入内容、工具选择和引用格式。

### 代码与结果

写代码时要贴近周围代码风格。不可逆或对外可见的动作要先确认，除非已经明确授权。发送到外部服务的内容可能被缓存或索引。删除或覆盖前要先查看目标；如果发现目标和描述不一致，应该告诉用户，而不是继续执行。

PM 读法：这是高风险操作的产品策略。它可以被转译成权限弹窗、二次确认、审计日志和撤销机制。

### 记忆

Claude Code 使用文件化记忆。每条记忆是一份带 frontmatter 的 Markdown 文件，分为用户信息、反馈、项目上下文和参考资料。`MEMORY.md` 只是索引，不放完整内容。写入前要检查是否已有相似记忆，错误记忆要删除，代码库本身已经记录的事实不要重复保存。

PM 读法：记忆功能不是“什么都存”。高质量记忆需要类型、索引、去重、更新、删除和有效性验证。

### 环境与上下文

prompt 会注入当前工作目录、平台、shell、模型信息、产品形态和日期。对话变长时，系统会压缩上下文并在后续窗口继续提供摘要，所以 agent 不需要提前结束任务。

PM 读法：长任务能力不仅来自模型上下文长度，也来自产品级上下文管理和恢复机制。

## PM 应该学什么

- 把 prompt 当成“运行协议”，而不只是回答风格。
- 把权限拒绝视为正常状态，设计恢复路径。
- 把记忆设计成可维护的数据结构。
- 把不可逆动作和外部发布动作单独列为高风险工作流。
- 把上下文压缩、trace、system reminder 看成 harness 责任。

## 设计练习

为自己的 agent 产品写一段 `Harness`：

- 用户看到的输出在哪里渲染？
- 工具失败、权限拒绝、网络失败分别如何处理？
- 哪些上下文是系统注入，不能当成用户指令？
- 结果里的文件、任务、媒体、链接应该用什么格式引用？
