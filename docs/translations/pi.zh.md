# Pi 0.79.3 中文意译

来源：[Phistory / Pi 0.79.3](https://phistory.cc/captures/pi/0.79.3/prompt.md)

## 一句话定位

Pi 的 prompt 很短，适合作为“最小 coding agent harness”样本：角色、工具、项目文档路径和基础文件操作。

## 核心汉化

### 角色

你是在 Pi 这个 coding agent harness 中运行的专家编码助手。你可以读文件、执行命令、编辑代码、写新文件。

PM 读法：这是最小角色定义。它没有复杂人格，也没有长工作流，但说明了最核心能力。

### 工具

Pi 提供 bash、edit、read、write 等基本工具，并提示不同项目可能有额外自定义工具。

PM 读法：最小 agent 也需要工具边界。用户和模型都要知道可操作范围。

### 文档

当用户询问 Pi 本身、SDK、扩展、主题、技能或 TUI 时，再读取相关文档路径。

PM 读法：这是一条上下文节约策略。只有在相关时才加载文档，而不是默认把所有说明塞进上下文。

## PM 应该学什么

- 极简 prompt 适合起步，但难以覆盖复杂安全和协作场景。
- 最小 harness 应至少说明角色、工具、文档和工作目录。
- 文档按需加载是控制上下文成本的好方法。

## 设计练习

写一个 20 行以内的最小 agent prompt：

- 它是谁？
- 它能做什么？
- 它有什么工具？
- 什么时候应该读取额外文档？
- 哪些事不能做？
