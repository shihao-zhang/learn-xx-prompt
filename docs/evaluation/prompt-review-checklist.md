# Prompt Review Checklist

用于评审一个 agent prompt 是否已经具备产品级可用性。

## 基础定位

- [ ] 是否说明 agent 是谁？
- [ ] 是否说明目标用户和任务范围？
- [ ] 是否说明不能做什么？
- [ ] 是否有安全或合规边界？

## Harness

- [ ] 是否说明输出在哪里渲染？
- [ ] 是否说明系统注入上下文和用户消息的区别？
- [ ] 是否说明工具失败或权限拒绝后的行为？
- [ ] 是否说明文件、图片、链接、任务状态等输出格式？

## 工具

- [ ] 是否列出真实可用工具？
- [ ] 是否说明何时优先使用专用工具？
- [ ] 是否说明高风险工具的确认条件？
- [ ] 是否说明并行、后台、长任务工具的使用边界？

## 工作流

- [ ] 是否定义简单任务和复杂任务的不同处理方式？
- [ ] 是否定义何时计划、何时执行、何时询问？
- [ ] 是否要求修改前先观察上下文？
- [ ] 是否要求完成前验证？

## 记忆与上下文

- [ ] 是否说明记忆写入条件？
- [ ] 是否说明不应保存什么？
- [ ] 是否有去重、更新和删除规则？
- [ ] 是否要求验证旧记忆仍然有效？

## 交付

- [ ] 是否要求说明改了什么？
- [ ] 是否要求说明如何验证？
- [ ] 是否要求如实报告失败或跳过的检查？
- [ ] 是否限制冗长日志和无关细节？
